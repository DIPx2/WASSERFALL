Структура данных и связи.

Система использует реляционную модель в SQLite для управления конфигурациями.
Конфигурационная БД (wasserfall_config.db):
hosts: Реестр серверов с флагом toggle для включения/исключения из рассылки.
Важно: Используется составной ключ UNIQUE(id_host, variable), что позволяет разным серверам иметь одинаковые имена переменных (например, свой PG_DB_PORT) без конфликтов.
sql_commands: Справочник SQL-шаблонов с поддержкой меток Jinja2 (например, {{ limit_val }}).
БД Логирования (wasserfall_logger.db):
execution_tasks: Заголовок задачи (целевой хост, итоговый SQL запрос).
execution_results: Результат (включая pg_code, системный exit_code и тело ответа в JSON).
